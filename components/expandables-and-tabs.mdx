### Troubleshooting guides

Organize solutions by symptom or error type:

<ExpandableGroup>
<Expandable title="üîç Login and Authentication Issues" default-open="false">

**"Invalid credentials" error**:

*Symptoms*: Login form shows "Invalid email or password" message

*Causes & Solutions*:
1. **Incorrect password** - Use the "Forgot Password" link to reset
2. **Account not verified** - Check your email for verification link
3. **Caps lock enabled** - Passwords are case-sensitive
4. **Corporate firewall** - Contact IT to allowlist `*.acme.com`

**Two-factor authentication problems**:

*Symptoms*: 2FA code not working or app not generating codes

*Solutions*:
1. **Time sync issues** - Ensure your phone's time is accurate
2. **Wrong authenticator app** - Make sure you're using the correct app
3. **Backup codes** - Use one of your saved backup codes
4. **Contact support** - We can temporarily disable 2FA to regain access

</Expandable>

<Expandable title="‚ö° Performance and Loading Issues" default-open="false">

**Slow page loading**:

*Symptoms*: Pages take more than 5 seconds to load

*Troubleshooting steps*:
1. **Check internet connection** - Try loading other websites
2. **Clear browser cache** - Ctrl+Shift+Delete (Windows) or Cmd+Shift+Delete (Mac)
3. **Disable browser extensions** - Try loading in incognito/private mode
4. **Check our status page** - Visit [status.acme.com](https://status.acme.com)

**App not responding**:

*Symptoms*: Interface freezes or buttons don't work

*Solutions*:
1. **Refresh the page** - F5 or Cmd+R
2. **Update your browser** - We support the latest 2 versions of major browsers
3. **Check JavaScript** - Ensure JavaScript is enabled in your browser
4. **Try different browser** - Test with Chrome, Firefox, or Safari

</Expandable>

<Expandable title="üîß Integration and Sync Issues" default-open="false">

**Slack notifications not working**:

*Check these settings*:
1. **Integration status** - Go to Settings ‚Üí Integrations and verify Slack shows "Connected"
2. **Channel permissions** - Ensure the bot has permission to post in your channels
3. **Notification preferences** - Check that relevant notifications are enabled
4. **Test the integration** - Use the "Test Connection" button in integration settings

**GitHub sync failing**:

*Common causes*:
1. **Token expired** - Refresh the GitHub integration in settings
2. **Repository permissions** - Ensure our app has access to the correct repositories
3. **Webhook configuration** - Check that webhooks are properly configured in GitHub
4. **Branch protection** - Verify our bot can create branches and pull requests

</Expandable>

## Best Practices

### Content organization strategy

**Logical grouping**:
- Group related topics together in ExpandableGroups
- Use descriptive titles that clearly indicate what's inside
- Order items from most common to least common
- Consider your users' mental model and workflow

**Progressive disclosure**:
- Start with essential information visible
- Use `default-open="true"` sparingly for critical items
- Hide advanced or detailed information in expandables
- Provide clear section previews in titles

### Writing effective expandable titles

**Good title examples**:
- ‚úÖ "How do I reset my password?" (clear question)
- ‚úÖ "Setting up GitHub integration" (specific task)  
- ‚úÖ "Troubleshooting login errors" (clear scope)
- ‚úÖ "API rate limits and quotas" (specific topic)

**Poor title examples**:
- ‚ùå "More information" (vague)
- ‚ùå "Click here" (non-descriptive)
- ‚ùå "Advanced" (unclear what's advanced)
- ‚ùå "Other stuff" (meaningless)

### When to use expandables vs. separate pages

**Use expandables for**:
- FAQ sections with short answers
- Optional details that supplement main content  
- Step-by-step processes with logical breaks
- Troubleshooting guides with multiple solutions
- Supplementary information that not everyone needs

**Use separate pages for**:
- Complete topics that deserve dedicated space
- Content that needs its own URL for linking/bookmarking
- Long-form guides or tutorials
- Content that forms part of a logical navigation flow
- Information that needs to rank in search results

<Callout kind="tip">
  **Accessibility consideration**: Expandable components are fully keyboard accessible and work with screen readers. The expand/collapse state is announced to assistive technologies.
</Callout>

curl -H "X-API-Key: YOUR_API_KEY" \

- 1000 requests per hour for authenticated requests
